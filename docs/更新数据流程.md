# 更新数据流程

## 适用场景

同事拿到新营期 Excel、CSV 或补充数据后，需要把看板更新成新数据。

## 推荐流程

1. 先把新数据整理成 `data/source_samples/` 里的模板结构。
2. 更新或生成 `data/runtime/dashboard_data_v1_5.json`。
3. 同步更新 `data/runtime/source_map_v1_5.md`，写清每个指标来源和公式。
4. 同步更新 `data/runtime/assumption_registry_v1_5.json`，把仍然是假设或推算的数据登记进去。
5. 运行构建脚本。
6. 运行 QA 脚本。
7. 打开 `index.html` 做人工验收。

## 命令

```bash
python3 scripts/build_v1_5_dashboard.py
python3 -m unittest qa/test_v1_5_2_ui.py -v
node qa/validate_v1_5_2_quota_funnel.cjs
```

## 不要做

- 不要只改 HTML 里已经嵌入的数据，而不保留源 JSON。
- 不要把假设数据直接写成真实数据。
- 不要删掉 `source_map_v1_5.md` 和 `assumption_registry_v1_5.json`。
- 不要把新营期原始表格直接塞进页面，先结构化再接入。
