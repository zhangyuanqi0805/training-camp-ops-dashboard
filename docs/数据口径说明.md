# 数据口径说明

## 当前数据分层

`data/runtime/` 是当前看板实际运行使用的数据：

- `dashboard_data_v1_5.json`：页面嵌入用主数据包。
- `source_map_v1_5.md`：指标来源、公式和边界。
- `assumption_registry_v1_5.json`：假设、推算、占位数据登记。

`data/source_samples/` 是给同事理解和后续更新用的样例数据：

- `602_structured_v1/`：602 期第一批结构化快照。
- `602_structured_v2/`：602 期补充结构化数据。
- `first_dashboard_model/`：第一版看板模型 CSV。
- `import_templates/`：后续导入模板。
- `metric_dictionary/`：字段和指标说明。

## 真实、计算、假设的区别

- 真实数据：来自已解析表格、结构化 CSV 或明确来源图谱。
- 计算数据：用真实字段按公式计算，例如 LTV、CAC、目标达成率。
- 假设数据：为了页面功能完整先补的样例或推算值，必须以 `assumption_registry_v1_5.json` 为准。

## 不应误读的点

- 首页漏斗是「每层人数占总配额比例」，不是逐层转化。
- 601-607 中部分营期为演示/假设数据，用于测试营期切换效果。
- 成本、CAC、ROI 等数据必须结合来源图谱和假设登记判断能否对外使用。
- 页面展示的是经营看板功能样例，不等同于最终正式财务报表。

## 正式使用前最少校验

1. 核对当前营期总人数、总配额和班级数。
2. 核对营收、LTV、成本、CAC 的来源表和时间范围。
3. 核对每日目标、实际、达成率和同比口径。
4. 核对假设数据是否已替换为真实数据。
5. 重新运行 `qa/` 下的验收脚本。
