# 营期经营驾驶舱资料包结构

用途：老大或运营把营期数据按这个结构给技术部，小内 / 小数就能刷新营期经营驾驶舱。

## 最小可刷新资料

先给 3 张必需表：

1. `01_learning_daily.csv`：学习过程，刷新到课率、完课率、回看率。
2. `03_conversion.csv`：成交与进阶，刷新真实 GMV、LTV、进阶率。
3. `04_cost.csv`：成本，刷新真实 CAC、ROI、LTV:CAC。

如果只能先给两张，优先给 `03_conversion.csv` 和 `04_cost.csv`，因为它们决定真实 LTV/CAC/ROI 能不能成立。

## 可选资料

- `02_channel_learning.csv`：渠道学习表现。
- `05_class_performance.csv`：班级与班班表现。
- `00_metadata.csv`：营期名称、数据截止时间、来源说明。

## 排版要求

- 第一行必须是字段名。
- 不要合并单元格。
- 不要多层表头。
- 一行一个观察对象：D 日、渠道、班级或成本记录。
- 金额字段只填数字，例如 `18800`，不要写 `¥18,800`。
- 百分比可以填 `0.242` 或 `24.2%`，技术部会统一解析。
- 缺失值留空或写 `missing`，不要用 `0` 冒充缺失。
- 真实值、历史参考、经营假设分开；假设值不能填进真实 GMV / 成本字段。

## 隐私边界

不要放手机号、微信号、openid、儿童姓名、付款明细、聊天原文。看板只需要聚合数据。
